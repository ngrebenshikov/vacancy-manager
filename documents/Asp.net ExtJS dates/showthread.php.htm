    aSP.NET AJAX DateTime (de)serialization

    aSP.NET AJAX DateTime (de)serialization


    For all of you that uses ASP.NET AJAX to do the serialization for you (and didn't see that the client-side code can be downloaded from ajax.asp.net until two days after you needed it and had to stay at work for far far too many hours):
    Code:
    var DeserializeJSON = function(data){
	// deserialize a Microsoft JSON date to a JavaScript date
	var exp = data.replace(new RegExp('(^|[^\\\\])\\"\\\\/Date\\((-?[0-9]+)\\)\\\\/\\"', 'g'), "$1new Date($2)");	
	return eval("(" + exp + ")");
};
    In Ext an example could be:
    Code:
    this.Connection = new Ext.data.Connection();
this.Connection.request({
	url: 'Events.asmx/GetWizardPlaceholder',
	params: { "eventid": row.data.eventID },
	method: 'POST',
	callback: function(options, success, response){
		this.Data = DeserializeJSON(response.responseText);
	},
	scope: Wizard
});

    GMT Offset correction

    GMT Offset correction


    I've been working with WCF and it has been serializing the GMT offset along with the time as follows:

    "\/Date(1225926000000-0700)\/"

    So here's a correction to your regex that should help account for that:

    Code:
    var DeserializeJSON = function(data){
	// deserialize a Microsoft JSON date to a JavaScript date
	var exp = data.replace(new RegExp('\\"\\\\/Date\\((-?[0-9]+)([-+][0-9]+)?\\)\\\\/\\"', 'g'), "new Date($1)");	
	return ("(" + exp + ")");
};

    I've added an additional Format to the Date-Object, so that these MS-style Dates may easily be parsed by Date.parseDate() for example or for a field definition in a JSONStore:

    Code:
    Date.parseCodes.X = {
	g: 1,
	c: ["u = parseInt(parseInt(results[1], 10)/1000);",
        "var sn = results[2]?results[2].substring(0,1):'+';",
        "var hr = results[2]?(results[2].substring(1,3)*1 + Math.floor(results[2].substring(3,5) / 60)):0",
        "var mn = results[2]?(results[2].substring(3,5) % 60):0;",
        "var diff = hr*3600+mn*60;",
        "u = sn=='+'?u+diff:d-diff;",
        "\n"
        ].join("\n"),
	s: "\/Date.([0-9]+)([+\-]\\d{4})?.+\/"
}
    So you can parse a Date using Date.parseDate("\/Date(1225926000000-0700)\/","X");
